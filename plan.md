# 썬드림 룰렛 이벤트 프로젝트 계획 및 기능 현황 (plan.md)

이 문서는 현재까지 개발된 썬드림 룰렛 이벤트 시스템의 주요 기능과 구조, 그리고 향후 계획을 정리한 문서입니다.

## 1. 프로젝트 주요 기능 현황 및 구현 로직

### 🎰 실시간 룰렛 엔진 (Roulette Engine)
- **서버 주도 당첨 방식**: 클라이언트의 조작을 방지하기 위해 서버(`comment_dart.py`)에서 최종 회전 각도와 당첨자를 미리 계산합니다.
  - **각도 랜덤화**: 720도에서 1440도 사이의 랜덤 각도를 생성하여 최소 2바퀴 이상의 회전을 보장합니다.
  - **상대 각도 계산**: 화살표가 가리키는 12시 방향(0도)을 기준으로, 원판의 회전 각도를 역산하여 `relative_angle = (360 - (final_angle % 360)) % 360` 공식을 적용합니다.
  - **섹터 매핑**: 모든 참여자를 가나다순으로 정렬한 뒤, 각자의 티켓 가중치에 비례하여 360도 공간 내 점유 섹터를 할당합니다.
- **애니메이션 동기화**: 서버는 모든 클라이언트에게 `start_game` 이벤트를 통해 회전 지속 시간(duration), 최종 각도(finalAngle), 당첨자를 전송하며, 모든 사용자는 소수점 단위까지 일치하는 물리적 회전을 목격합니다.

### 🔗 네이버 카페 연동 및 상호작용 (Scraping Strategy)
- **다중 API 전략**: 네이버 카페의 V2 API를 활용하며, 페이지네이션, 단일 대량 요청, 커서(Cursor) 기반 수집 등 3가지 전략을 순차적으로 시도하여 데이터 누락을 최소화합니다.
- **점진적 수집 (Incremental Fetch)**: `last_comment_id`를 추적하여 새로운 댓글만 효율적으로 가져오며, 서버 부하를 줄입니다.
- **Selenium 폴백 (Fallback)**: API 호출이 제한되거나 정밀한 DOM 수집이 필요한 경우, ChromeDriver 기반의 `SeleniumCommentScraper`가 실제 브라우저를 구동하여 댓글을 가로챕니다.
- **화이트리스트 처리**: 수집된 댓글 작성자가 `allowed_list`에 포함된 경우에만 룰렛 참여자로 승격시키며, 설정된 티켓 수만큼 확률 가중치를 부여합니다.

### ☁️ 데이터 동기화 및 안정성 (DB Sync & Persistence)
- **하이브리드 DB 아키텍처**: 클라우드(Supabase)와 로컬(SQLite)을 동시에 운영하여 속도와 데이터 보존성을 모두 확보합니다.
  - **하이드레이션 (Hydration)**: 서버 시작 시 Supabase의 최신 데이터를 로컬 SQLite에 복제하여 초기 로딩 속도를 극대화합니다.
  - **비동기 업서트 (Async Upsert)**: 로컬 DB에 데이터를 저장한 뒤, 사용자 체감 지연이 없도록 백그라운드 스레드에서 Supabase로 데이터를 실시간 동기화합니다.
- **실시간 상태 감지 (2s Polling)**: `monitor_view.py`의 폴링 루프가 2초 간격으로 Supabase의 변경 사항(경품 변경, 당첨자 추가 등)을 감지하여 접속 중인 모든 클라이언트의 UI를 즉시 갱신합니다.

### 🎨 인터페이스 및 상태 관리 (UI/UX Logic)
- **Socket.IO 실시간 통신**: 모든 게임 명령(Reset, Start, Winner Confirm)과 소리 재생(Beep, Fanfare)은 전용 소켓 이벤트를 통해 즉각적으로 전달됩니다.
- **10초 쿨다운 로직 (Cooldown)**: 서버 사이드에서 `last_winner_confirm_times`를 기록하여, 당첨자 발표 후 10초 이내에는 새로운 슈팅 시간을 설정할 수 없도록 강제 방어합니다.
- **동적 가나다 정렬**: 화살표의 위치와 섹터의 순서가 모든 화면에서 동일하도록, 참가자 명단은 항상 서버와 클라이언트 모두에서 엄격한 가나다순 정렬을 유지합니다.

## 2. 기술 스택
- **Backend**: Python, Flask, Flask-SocketIO
- **Frontend**: HTML5, Vanilla CSS, JavaScript, Chart.js
- **Database**: Supabase (PostgreSQL), SQLite
- **Automation**: Selenium (Chrome WebDriver)

## 3. 설치 및 실행 요약
1. `pip install -r requirements.txt`
2. `python comment_dart.py` 실행 (기본 5000 포트)
3. `/monitor_page` 에서 이벤트 URL 입력 및 설정
4. `/` 또는 `/guest` 페이지에서 룰렛 진행
