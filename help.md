# 룰렛 프로젝트 인수인계 메모 (2026-04-09 리셋본)

## 1) 현재 상황 요약

- 최근 세션에서 운영 UX(새 ID, 종료/취소, 당첨 체크, 시간 버튼 피드백) 위주로 다수 수정됨.
- 다수 문제는 해결됨(아래 "완료된 변경").
- 그러나 **`새 이벤트로 등록` 직후 `(NEW)` 상태가 예고 없이 사라져 "저장된 것처럼 보이는" 핵심 이슈는 재발 보고됨**.
- 사용자 판단: 이슈가 누적되어 새 세션에서 깨끗하게 다시 진행 필요.

## 2) 가장 급한 미해결 이슈 (P0)

### P0-1. NEW 표시가 안정적으로 유지되지 않음

#### 현상
- `새 이벤트로 등록` 후 `(NEW)`가 잠깐 보이다가 사라지는 경우가 있음.
- 실제로는 저장 버튼을 누르지 않았는데 저장된 것처럼 보임.
- 사용자 혼란 포인트:
  - "지금 draft인가, 저장된 이벤트인가?" 구분 불가
  - 히스토리/제목/ID 배지의 상태가 순간적으로 엇갈림

#### 현재 구현 상태
- NEW 배지/상태 관련 로직이 다층으로 존재:
  - `_opRecentResetKeys` (TTL 기반)
  - `_opDraftEventMeta` (draft key 기반)
  - `_opStickyNewKey` (고정키 기반)
  - 제목 접두 `(NEW)` 방식 추가
- 저장 전/후, 소켓 동기화, `select` 응답, draft 모드 전환이 섞이며 상태 경합 가능성 높음.

#### 다음 세션 권장 해법 (강력 권장)
- **NEW 표시 단일 소스로 단순화**
  - 권장: `window._opStickyNewKey` 하나만 신뢰
  - 제목 접두 `(NEW)`도 같은 소스에서만 결정
- **저장 여부 판정 단일화**
  - NEW 해제는 오직 "현재 key의 수동 저장 성공 응답"에서만
  - 그 외 소켓/비동기 응답으로는 NEW 상태 변경 금지
- **stale 응답 무시 강화**
  - `save/select` 응답의 `event_key != 현재 폼 key`이면 즉시 discard

## 3) 완료된 변경 (이미 반영/푸시됨)

### 운영 UX/동작
- 확인창(confirm) 제거, 토스트/상태 메시지 유지
- 저장 직후 라이브 패널 즉시 반영(제목만 반영되는 문제 개선)
- 시간 `+60초/+50초` 클릭 시 미세 토스트 + 입력창 펄스 효과 추가
- 사은품 "추첨 중" 박스 오버플로우 개선(가로 스크롤 이슈 완화)
- 당첨 직후 체크 해제 보류(다음 시작 시점에 반영)

### 사운드/타이밍
- 팡파레 중복 재생 방지(라운드당 1회 가드)
- 정지 타이밍 보정값 미세 조정

### 종료/취소
- 제목 앞 `(종료)` 태그 표시
- 종료 상태를 DB에 영속화(새로고침 후 유지)
- 종료/취소 시 제목/히스토리 즉시 반영(새로고침 없이)

## 5) 새 세션에서 바로 할 작업 순서 (실행형)

### Step 1) NEW 상태 로직 단순화
- 파일: `templates/index.html`
- 목표:
  - NEW 판단 함수 1개로 통일
  - 배지/히스토리/제목 접두가 동일한 진실 소스를 보게 정리
- 권장:
  - `opIsNewKey()` 내부를 `_opStickyNewKey` 중심으로 축소
  - TTL/draft 보조 로직은 최소화하거나 제거

### Step 2) 저장/선택 응답 경합 차단
- `opSave`/`opRegisterNew`/`opApplySelectPayload`에서
  - 현재 key와 응답 key mismatch 시 하드 discard
  - draft 모드일 때 비의도 `select payload`는 절대 적용 금지

### Step 3) 재현 테스트 (필수)
- 시나리오 A:
  1) 기존 이벤트 선택
  2) `새 이벤트로 등록`
  3) 제목 1글자 입력
  4) 5~10초 대기
  5) `(NEW)` 유지 여부 확인
- 시나리오 B:
  1) 위 상태에서 저장
  2) 저장 성공 직후 `(NEW)` 제거 확인
  3) 새로고침 후 상태 일치 확인
- 시나리오 C:
  1) `룰렛 종료` 클릭
  2) 새로고침
  3) `(종료)` 및 버튼 비활성 유지 확인
  4) `종료 취소` 후 즉시 반영/새로고침 반영 확인
